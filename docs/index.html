Getting Started

Lumi UI is a lightweight, declarative UI components library for creating SPAs (Single Page Applications).

Lumi takes the simple approach that can be summarized in the following points:

  • Each UI element is declared by a plain JS object
  • Listener callbacks are handled through normal JavaScript functions
  • Component access is handled using an unique id for each UI component and a global accessor function $$('id')
This is not an innovative framework, nor does it try to be. If you're after something more progressive, try something like Vue or React. Rather, Lumi tries an arguably simpler and more straight-forward that's been around for quite some time.

Installation & Setup

Lumi can be installed using npm or bower:

bower install lumi

npm install lumi

To add styling, add this tag to your HTML file:

<link rel="stylesheet" href="css/lumi.css">

To use include the icon files, also add:

<link rel="stylesheet" href="css/lumi-icon.css">

For development, add the following script files:

<script src="jquery.min.js" type="text/javascript"></script>
<script src="lumi.min.js" type="text/javascript"></script>

For release, use:

<script src="jquery.js" type="text/javascript"></script>
<script src="lumi.debug.js" type="text/javascript"></script>

More Ramblings...

The problem with many similar object-centric UI frameworks is the need to relearn mappings of property names to functionality resulting in the need to constantly revisit the documentation. To simplify this process, Lumi tries to resemble traditional HTML tags and CSS properties as much as possible.

Where this approach deviates from the traditional approach is with the layout system. Here, Lumi takes a simpler and more modern approach with the flexbox system as opposed to a grid system, as seen by popular frameworks like Bootstrap. The drawback to this, is older browsers (mainly IE) do not support this.

Example Usage

var modal = UI.new({
  "view": "modal",
  "header": {
    "view": "label",
    "htmlTag": "H3",  // Optional, default is "SPAN"
    "label": "Lumi is exquisite!"
  },
  "body": {
    "view": "label",
    "label": "Hello..."
  },
  "footer": {
    "flexAlign": "right",
    "cells": [
      {
        "view": "button",
        "label": "No way!",
        "margin": "right"
      },
      {
        "view": "button",
        "type": "primary",
        "label": "Yup."
      }
    ]
  }
}, document.body);

// Opens the modal
modal.open();

Extensible

Brand new Lumi component templates can also be defined using JavaScript objects. This enables the ability to define new components such as switchers, drawers, chips, etc.

To understand how to create your own custom components, let's take a look at an example from the code below:

UI.def({
  __name__: "toggle",
  $setters: extend(
    classSetters({
      color: prefixClassOptions({
        "success": "",
        "danger": "",
        "warning": "",
        "": ""
      }, 'uk-toggle-', true)
    }),
    {
      checked: function (value) {
        this.getFormControl().checked = value;
      }
    }
  ),
  $defaults: {
    htmlTag: "LABEL",
    tagClass: "uk-toggle",
    type: 'checkbox'
  },
  template: '<input><div class="uk-toggle-slider"></div>',
  getFormControl: function () {
    /**
     * Get the HTML input element.
     * @returns {Element}
     */
    return this.el.firstChild;
  },
  reset: function () {
    /**
     * Reset the toggle.
     */
    this.getFormControl().checked = false;
  },
  getValue: function () {
    /**
     * Get the value of the toggle.
     * @returns {boolean}
     */
    return this.getFormControl().checked;
  },
  setValue: function (value) {
    /**
     * Set the value of the toggle.
     * @param value
     */
    this.getFormControl().checked = value;
  }
}, UI.ChangeEvent, UI.FormControl, UI.definitions.element);

Inheritance

Notice that you can chain previously defined components or Mixins at the end of the definition object: UI.def(config, ext1, ext2, ...), thus providing the ability to extend library components.

Note When overriding properties, the left-most object has the highest priority. In other words, each object will have its properties overridden by anything left of it.

Testing

Writing UI as ordinary JavaScript components actually makes things very testable. Lumi uses its own event-dispatching system and you can simulate button clicks, change events, or even custom events!

Below is an example of a Jasmine test case:

it("someToggle dispatch onChange event", function () {
  spyOn(self, "someAction");
  $$("someToggle").setData([
    {label: "someLabel1", value: 1},
    {label: "someLabel2", value: 2}
  ]);
  $$("someToggle").setValue(1);
  $$("someToggle").dispatch("onChange", []);
  expect(self.someAction).toHaveBeenCalledWith(1);
  $$("someToggle").setValue(2);
  $$("someToggle").dispatch("onChange", []);
  // And so on...
});
